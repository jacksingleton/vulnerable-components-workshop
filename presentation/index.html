Using Components with Known Vulnerabilities

  • Prevalence: WIDESPREAD
  • Detectability: DIFFICULT

So what do we have to lose?

  • Confidentiality
  • Integrity
  • Availability
  • Core app sec terminology
  • A vulnerability in a component can turn out to be anything, often even remote code execution!

When should I be careful?

  • Including a new component
  • Finding a component you didn't know about
  • Picking up legacy applications

Secure Code Checklist

  • Keep components up to date
  • Know what components and versions your application uses
  • Evaluate components before including
